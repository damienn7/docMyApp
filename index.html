Installation de Flutter et Dart

1. Pourquoi Dart ?

  • Performance : Dart compile directement en code machine natif, ce qui permet des performances élevées et une exécution rapide des applications.
  • Productivité : Dart offre des fonctionnalités comme le hot reload, permettant de voir instantanément les changements dans l'application sans devoir la redémarrer.
  • Facilité d'apprentissage : Dart est relativement facile à apprendre, surtout pour ceux qui ont une expérience avec des langages de programmation comme JavaScript ou Java.

2. Utiliser Flutter avec Dart

Télécharger et Installer Flutter SDK :

sh
                        cd ~
                        curl -O https://storage.googleapis.com/flutter_infra_release/releases/stable/linux/flutter_linux_3.0.0-stable.tar.xz
                        tar xf flutter_linux_3.0.0-stable.tar.xz
                        echo 'export PATH="$PATH:$HOME/flutter/bin"' >> ~/.bashrc
                        source ~/.bashrc

Vérifier l'installation de Flutter :

sh
                        flutter doctor

Configurer Android Studio

Installer Android Studio sur Windows :

Téléchargez et installez Android Studio depuis developer.android.com/studio.

Configurer les Outils de Développement Android :

Ouvrez Android Studio, allez dans Configuration > SDK Manager et installez les SDKs nécessaires pour Android.

Configurer les Variables d'Environnement pour WSL :

sh
                        echo 'export ANDROID_HOME=/mnt/c/Users/<your-username>/AppData/Local/Android/Sdk' >> ~/.bashrc
                        echo 'export PATH=$PATH:$ANDROID_HOME/tools/bin' >> ~/.bashrc
                        echo 'export PATH=$PATH:$ANDROID_HOME/platform-tools' >> ~/.bashrc
                        echo 'export PATH=$PATH:$ANDROID_HOME/emulator' >> ~/.bashrc
                        source ~/.bashrc

Configurer un Émulateur Android

Créer un Émulateur Android :

Depuis Android Studio, utilisez l'AVD Manager pour créer un nouvel émulateur Android.

Lancer l'Émulateur :

sh
                        emulator -avd <name-of-your-emulator>

Créer et Exécuter un Projet Flutter

Créer un Nouveau Projet Flutter :

sh
                                flutter create my_app
                                cd my_app

Exécuter le Projet Flutter :

sh
                        flutter run

4. Intégration avec API Publique et Fonctions du Téléphone

Intégrer une API Publique :

Utilisez le package http pour effectuer des requêtes HTTP.

Ajoutez le package à votre pubspec.yaml :

dependencies:
                          flutter:
                            sdk: flutter
                          http: ^0.13.3

Utiliser les fonctionnalités du téléphone :

Utilisez des packages comme geolocator pour accéder à la localisation, sensor pour l'accéléromètre et le gyroscope, etc.

Exemple d'utilisation de geolocator :

dependencies:
                          geolocator: ^8.0.0

Exemple d'utilisation dans votre code :

import 'package:geolocator/geolocator.dart';
                        
                        Future _getCurrentLocation() async {
                          Position position = await Geolocator.getCurrentPosition(desiredAccuracy: LocationAccuracy.high);
                          print(position);
                        }

5. Avantages de Flutter et Dart

  • Développement Rapide :
    • Hot Reload : Permet de voir les changements instantanément.
    • Widgets : Une large bibliothèque de widgets préconstruits pour créer des interfaces utilisateur attractives.
  • Performance Elevée :
    • Compilation Natif : Dart compile en code natif, offrant des performances similaires à celles des applications écrites en Swift ou Kotlin.
  • Support Multi-plateformes :
    • Créez des applications pour Android, iOS, Web et Desktop avec une seule base de code.

Plan

1. Technos Possibles

1.1 Frameworks

  • Flutter: Développé par Google, Flutter permet de créer des applications nativement compilées pour mobile, web et desktop à partir d'une base de code unique.
  • React Native: Développé par Facebook, ce framework permet de construire des applications mobiles en utilisant JavaScript et React.
  • Ionic: Framework open-source pour la création d'applications mobiles hybrides en utilisant HTML, CSS et JavaScript, souvent avec Angular.
  • Kotlin: Langage de programmation moderne, sécurisé et concis pour le développement d'applications Android natives.

1.2 Environnements de Développement

  • Android Studio: IDE officiel pour le développement Android, supportant Java, Kotlin et Flutter.
  • Xcode: IDE officiel pour le développement iOS, supportant Swift et Objective-C.
  • Visual Studio Code: IDE polyvalent, utilisé souvent avec Flutter et React Native grâce à ses extensions.

1.3 API Management

  • Retrofit: Librairie pour Android et Java pour simplifier les appels API REST.
  • Axios: Client HTTP basé sur les promesses pour JavaScript, utilisé souvent avec React Native.
  • Dio: Puissant client HTTP pour Dart, souvent utilisé avec Flutter.

2. Techno Conseillée

Flutter est recommandé pour plusieurs raisons :

  • Cross-Platform: Une base de code unique pour Android et iOS.
  • Performances: Utilise Dart qui compile en code natif, offrant des performances proches du natif.
  • UI Riches: Large éventail de widgets personnalisables et out-of-the-box.
  • Communauté: Forte communauté de développeurs et support de Google.
  • Documentation: Documentation exhaustive et ressources abondantes pour les débutants.

3. Idées de Projet Mobile Accessible pour un Niveau Intermédiaire

  • Application de Fitness: Utilise l'accéléromètre et le gyroscope pour suivre les mouvements et la position lors des exercices, combiné avec une API de nutrition pour fournir des conseils diététiques.
  • Guide de Voyage: Utilise les capteurs de proximité et l'orientation pour offrir des informations sur les points d'intérêt à proximité, en se connectant à une API de guide de voyage.
  • Gestion de Tâches et de Projets: Intégration avec l'agenda et les contacts pour gérer les tâches et projets collaboratifs, en utilisant une API de gestion de tâches.

4. Idée de Projet Conseillée

Application de Fitness :

  • Fonctionnalités: Suivi des mouvements, suivi de la position, conseils diététiques.
  • Capteurs Utilisés: Accéléromètre, gyroscope, GPS.
  • API Utilisée: API de nutrition (ex: Edamam, Nutritionix).

5. Détails du Sujet avec Points Clés

Nom de l'application: FitTrack

Fonctionnalités Clés:

  • Suivi d'Activité: Utilisation de l'accéléromètre et du gyroscope pour suivre les mouvements pendant l'exercice.
  • Suivi de Position: Utilisation du GPS pour suivre les parcours de course ou de marche.
  • Conseils Nutritionnels: Connexion à une API de nutrition pour fournir des recommandations diététiques personnalisées.
  • Historique et Statistiques: Suivi des progrès et des performances au fil du temps.

Requêtes API:

  • Recherche d'Aliments: Obtenir des informations nutritionnelles sur des aliments spécifiques.
  • Recommandations de Recettes: Rechercher des recettes saines basées sur les préférences alimentaires.
  • Suivi des Calories: Suivre l'apport calorique quotidien.
  • Suivi des Nutriments: Obtenir des informations sur les macros et micronutriments consommés.
  • Analyse des Repas: Analyse des repas basés sur les informations saisies par l'utilisateur.

Capteurs Utilisés:

  • Accéléromètre: Suivi des mouvements et des activités physiques.
  • Gyroscope: Détection des orientations et des changements de posture.
  • GPS: Suivi des parcours et des distances parcourues.
